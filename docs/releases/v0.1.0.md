# v0.1.0 — Public OSS workflow baseline

This release note is prepared for the first public baseline release of `ai-devos-kit`.

## Release scope

Included baseline surfaces:

- local-first AI-agent workflow kit README;
- MIT license;
- contribution guide;
- security policy;
- pull request template;
- public-safety validator;
- GitHub Actions public-safety workflow;
- existing local workflow scripts and templates for repository health, context capture, handoff, and agent run artifacts.

## Validation before publishing

Run from the repository root:

```bash
python3 scripts/check_public_safety.py
git diff --check
git status --short
```

Environment-specific scripts may also be run when available:

```bash
bash scripts/repo_health_check.sh
bash scripts/secret_redacted_scan.sh
```

Do not claim these environment-specific checks passed unless their exact output is captured.

## Publishing path

After local `main` is synced to the validated remote HEAD:

```bash
git tag -a v0.1.0 -m "v0.1.0 — Public OSS workflow baseline"
git push origin v0.1.0
```

Manual release fallback:

```bash
gh release create v0.1.0 \
  --repo EthanSangSSS/ai-devos-kit \
  --title "v0.1.0 — Public OSS workflow baseline" \
  --notes-file docs/releases/v0.1.0.md \
  --verify-tag
```

## Honesty boundary

This release does not claim stars, downloads, production adoption, external users, or OpenAI endorsement. It is an early-stage public baseline for reusable, evidence-bounded AI-agent development workflows.

## Known limitations

- The public-safety scan is lightweight and dependency-free; it does not replace full secret scanning or security review.
- Local scripts may be environment-specific and should not be treated as universally portable until validated in a target project.
- Human review remains required before applying generated templates, prompts, or automation to production repositories.
